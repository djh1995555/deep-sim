# 目标
基于深度学习方法训练一个车辆高保真动力学模型用于仿真系统，能够仿真车辆在干燥以及湿滑雨雪路面上的动力学响应，不是用来做实时车辆控制的。
这是一个工程落地，不是为了发论文，创新不重要，重要的是效果

# 要求
- 调研50篇论文
- 交流和输出文档都是中文

# **难点**
- 如何让模型具有泛化性，能自适应不同的路面
- 如何考虑同一时刻，各个车轮在不同路面上的情况（Split-μ）
- 如何处理车辆在不同路面衔接处的表现
- 轮胎小角度和大角度（非线性区）的差异大如何处理
- 常规路况下输入的信息非常稀疏，如何处理
- 如何从车辆历史状态信息中提取到路面的信息，分析可辨识性，辨识不到的部分如何处理
- 如何计算模型输出结果的置信度
- 物理约束如何加入

# 输入
## 车辆可观测状态
以下是真实车辆上能够通过传感器测量得到的信号，也作为整个车辆动力学模型的输入，也可以以此来确定方案用采用的车辆动力学物理模型自由度

| 量名 | 物理含义 | 在系统中的用途 |
| --- | --- | --- |
| `vx` | 车体坐标系下的纵向速度，表示车辆沿车头方向的运动速度 | backbone 主状态，决定纵向动力学、滑移和 rollout 主链路 |
| `vy` | 车体坐标系下的横向速度，表示车辆相对车身坐标系的侧向运动速度 | backbone 主状态，决定横向动力学、侧偏和横摆响应 |
| `roll` | 车身绕纵轴的横滚角 | 反映横向载荷转移和车身侧倾 |
| `pitch` | 车身绕横轴的俯仰角 | 反映加速/制动引起的纵向载荷转移 |
| `yaw` | 车身绕垂向轴的偏航角 | 表示车辆朝向变化 |
| `p` | 横滚角速度 | 描述车身侧倾变化速度 |
| `q` | 俯仰角速度 | 描述车身点头/抬头变化速度 |
| `r` | 偏航角速度 | 横向动力学核心状态之一，用于刻画转向响应 |
| `omega_fl, omega_fr, omega_rl, omega_rr` | 四个车轮的角速度 | 用于纵向滑移计算、轮胎纵向力计算和驱动/制动工况判断 |
| `sw_angle` | 方向盘角度，不等于前轮实际转角 | steering effect 模块核心输入，用于推断有效转向作用 |
| `tau_drv_fl/fr/rl/rr` | 四轮轮端驱动扭矩 | 作为轮胎纵向力生成的边界条件，反映驱动力输入 |
| `tau_brk_fl/fr/rl/rr` | 四轮轮端制动扭矩 | 作为轮胎纵向力生成的边界条件，反映制动力输入 |
| `steer_cmd` | 上层转向控制请求 | 描述控制器或驾驶员希望施加的转向命令，**传动比未知** |
| `throttle_cmd` | 上层油门请求 | 描述动力请求强度，不等于轮端实际驱动扭矩 |
| `brake_cmd` | 上层制动请求 | 描述制动请求强度，不等于轮端实际制动扭矩 |
## 视觉信息
第一阶段只采用车辆可观测信息，第二阶段可以接入视觉信息，可以给出当前处于什么状态的路面（湿滑,）
# 输出
训练得到的车辆动力学模型输出为长时多步预测状态值，计算rollout误差
# 数据
- 当前条件所限，所以准备第一版训练完全采用超高自由度车辆动力学模型来生成数据（模拟真实车辆采集数据，真实车辆上需要考虑的各种效应都可以加上，比如悬架特性，专项特性，轮胎特性，Pacejka、摩擦椭圆、水滑、冰面速度效应、动态载荷转移，传感器噪声，车辆震动），但是只有上述的可观测输入可以被用来训练，其他内部如悬架，轮胎等内部数据都不能参与训练，因为真实车辆上这些数据无法获取。
- 用来训练的车辆动力学模型物理基座采用的是低自由度模型（自由度取决于可获取的状态量数量）
- 数据要求覆盖常规稳态工况和低附着路面，极限操控工况和组合工况，但是第一版先做常规稳态工况和低附着路面专项,后续加入极限操控工况和组合工况
