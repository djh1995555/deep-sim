## 1. 固定前提

三组方案共享同一个物理 backbone：

```text
double-track 5-DOF body + 4-wheel rotational dynamics
+ roll/pitch dynamics
+ explicit Fz_i
+ S2 steering: first-order steering lag
+ wheel-level μ_mean/logvar
+ multi-step rollout loss
```

其中 `wheel-level μ_mean/logvar` 由 `MuHead` 输出，三组方案都保留 `MuHead`；三组差异只在 tire residual/global residual 的组织方式。

共享物理中间量：

```text
κ_i      slip ratio
α_i      slip angle
Fz_i     four-wheel normal load
μ_i      wheel-level road friction latent
Fx_i,Fy_i tire forces
delta_eff effective steering angle
```

第一版默认：

```text
Steering: S2
Vehicle residual placement: V1, residual added to x_dot before integration
```

S3 steering 和 V2 state residual 保留为对照。

## 2. 三组主实验方案

### Group 1: Physical Tire Model + Single Global Residual

一句话：轮胎力完全由物理轮胎模型计算，不单独设置 tire residual；只保留一个 residual 模型，负责覆盖轮胎误差和整车误差。

结构：

```text
Fz_i = LoadTransferPhysics(...) + optional bounded ΔFz_i

Fx_i, Fy_i = TirePhysics(κ_i, α_i, Fz_i, μ_i)

x_dot_phys = DoubleTrackRollPitchDynamics(x_t, Fx_i, Fy_i, delta_eff)

Δx_dot_global = GlobalResidualNN(h_t, x_t, u_t)

x_next = Integrator(x_t, x_dot_phys + Δx_dot_global)
```

`GlobalResidualNN` 负责覆盖：

```text
tire model mismatch
vehicle low-DOF model mismatch
actuator lag / steering residual not captured by S2
sensor/filter/discretization effects
```

优点：

- 模块最少，最容易实现和调试；
- 可以验证物理轮胎模型本身是否足够；
- residual 位置简单，适合作为 Rank 1 前的强 baseline。

风险：

- tire error 和 vehicle error 混在一个 residual 中，可解释性较弱；
- residual 可能吞掉轮胎物理模型；
- 对低附着/Split-μ 的 tire force 上限补偿不如显式 tire residual 精准。

最小实验：

```text
G1-V0: no residual
G1-V1: Δx_dot_global before integration
G1-V2: Δx_global after integration
```

成功信号：

- 相比 physics-only，5s rollout error 明显下降；
- residual 幅值受控；
- 低附着/Split-μ 下 constraint violation 不显著增加。

### Group 2: Force-Level Tire Residual

一句话：保留整车 residual，同时新增 tire residual；tire residual 输出 `ΔFx/ΔFy`，并投影到 friction ellipse。

结构：

```text
Fz_i = LoadTransferPhysics(...) + optional bounded ΔFz_i

Fx_phys_i, Fy_phys_i = TirePhysics(κ_i, α_i, Fz_i, μ_i)

ΔFx_i, ΔFy_i = TireResidualNN(κ_i, α_i, Fz_i, μ_i, h_t, wheel_id)

Fx_i, Fy_i = project_to_friction_ellipse(
  Fx_phys_i + ΔFx_i,
  Fy_phys_i + ΔFy_i,
  μ_i,
  Fz_i
)

x_dot_phys = DoubleTrackRollPitchDynamics(x_t, Fx_i, Fy_i, delta_eff)
Δx_dot_vehicle = VehicleResidualNN(h_t, x_t, u_t)
x_next = Integrator(x_t, x_dot_phys + Δx_dot_vehicle)
```

优点：

- tire error 和 vehicle error 分离；
- 对湿滑、雪地、Split-μ、轮胎非线性区更直接；
- friction ellipse projection 防止 tire residual 输出非法力。

风险：

- 模块更多，调试更复杂；
- `ΔFx/ΔFy` 如果过强，仍可能绕过物理 tire model；
- 需要 residual regularization 和 projection。

最小实验：

```text
G2: TirePhysics + ΔFx/ΔFy + projection + V1 VehicleResidualNN
```

成功信号：

- 相比 G1，在 wet/snow/Split-μ 中 `vy/r/omega_i` rollout error 下降；
- friction ellipse violation 接近 0；
- `VehicleResidualNN` 幅值小于 G1，说明 tire error 被更合理地吸收在 tire residual 中。

### Group 3: Parameter-Level Tire Residual

一句话：保留 tire residual，但模型不直接输出力，而是输出 `ΔC_alpha/ΔC_kappa/Δμ_scale` 等轮胎参数修正。

结构：

```text
Fz_i = LoadTransferPhysics(...) + optional bounded ΔFz_i

ΔC_alpha_i, ΔC_kappa_i, Δμ_scale_i =
  TireParamResidualNN(κ_i, α_i, Fz_i, μ_i, h_t, wheel_id)

θ_tire_eff_i = θ_tire_base_i + Δθ_tire_i
μ_eff_i = μ_i * clamp_positive(1 + Δμ_scale_i)

Fx_i, Fy_i = TirePhysics(κ_i, α_i, Fz_i, μ_eff_i, θ_tire_eff_i)

x_dot_phys = DoubleTrackRollPitchDynamics(x_t, Fx_i, Fy_i, delta_eff)
Δx_dot_vehicle = VehicleResidualNN(h_t, x_t, u_t)
x_next = Integrator(x_t, x_dot_phys + Δx_dot_vehicle)
```

优点：

- 比 force residual 更可解释；
- tire force 仍由物理模型生成，天然更受约束；
- 适合常规和中低 slip 区。

风险：

- 参数 residual 表达能力可能不足；
- `C_alpha/C_kappa/μ_scale` 在低附着、combined slip、大侧偏下可能强耦合；
- 参数含义可能随数据分布漂移。

最小实验：

```text
G3: TirePhysics with learned parameter residual + V1 VehicleResidualNN
```

成功信号：

- 常规 dry/wet 稳态场景接近或优于 G2；
- residual 更平滑、更小；
- 低附着极限区若明显弱于 G2，则 G3 降级为可解释对照。

## 3. 推荐排序

当前推荐执行顺序：

1. **Group 1**：先建立物理轮胎模型 + 单一 residual 的最小闭环。
2. **Group 2**：在 G1 基础上拆出 force-level tire residual，验证是否显著改善低附着和 Split-μ。
3. **Group 3**：用 parameter-level tire residual 对照 G2，评估可解释性与性能取舍。

预期最终主方案大概率是：

```text
Group 2 if low-μ/Split-μ improvement is significant
Group 3 if it matches G2 while residual smoother and more stable
Group 1 if tire physics is already strong enough and added tire residual收益不足
```

## 4. 共享 ablation

### SteeringHead

```text
S2: first-order steering lag                       # 默认
S3: S2 + bounded NN steering residual              # 对照
```

S0/S1 只作为简单参考基线，需要时再补。

### VehicleResidualNN placement

```text
V0: no vehicle/global residual
V1: x_next = Integrator(x_t, x_dot_phys + Δx_dot)   # 默认
V2: x_next = Integrator(x_t, x_dot_phys) + Δx       # 对照
V3: Δx_dot + Δx                                    # 暂不主推
```

### FzResidualNN

```text
F0: no FzResidualNN
F1: bounded FzResidualNN
F2: bounded FzResidualNN + Fz_true teacher loss, if simulator exports Fz_true_i
```

### Uncertainty

```text
U0: single model + heteroscedastic logvar
U1: K=3 deep ensemble
```

开发初期先 U0，模型稳定后再 U1。

## 6. Suggested Execution Order

1. 实现 physics-only double-track + roll/pitch + `Fz_i`，作为可运行 baseline。
2. 实现 S2 steering baseline。
3. 实现 Group 1：物理轮胎模型 + 单一 global residual，形成第一版 Rank 1 闭环。
4. 对 Group 1 做 V0/V1/V2 对照，默认保留 V1。
5. 实现 Group 2：新增 `ΔFx/ΔFy` tire residual + friction ellipse projection。
6. 实现 Group 3：新增 `ΔC_alpha/ΔC_kappa/Δμ_scale` parameter residual。
7. 加 FzResidualNN，并与 no-FzResidualNN 对照；若仿真器导出 `Fz_true_i`，加入 teacher loss 对照。
8. 加入 S3：bounded NN steering residual，并与 S2 对照。
9. 做 black-box baseline 对照。
10. 单模型稳定后再训练 K=3 ensemble。
11. 极限操控数据加入后再做 MoE tire residual。
